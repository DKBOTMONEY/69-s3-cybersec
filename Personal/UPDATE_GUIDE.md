# คู่มือการอัปเดตและสรุปงานปรับปรุงดีไซน์ (Web Design & Content Update Guide)

ไฟล์นี้สร้างขึ้นเพื่อสรุปรายการปรับปรุงดีไซน์ที่ได้ทำไป และเป็นคู่มือสำหรับการเข้าไปแก้ไข/อัปเดตข้อมูลเนื้อหาบนหน้าเว็บด้วยตนเองในอนาคตครับ

---

## 🛠️ 1. สรุปสิ่งที่ได้ทำไป (What Was Done)

เราได้ทำการเปลี่ยนโฉมการจัดวางและดีไซน์ (Visual Overhaul) โดยอ้างอิงต้นแบบและแนวทางจากโฟลเดอร์อ้างอิง ดังนี้:

*   **โทนสีหลัก (Theme Palette)**: เปลี่ยนเป็นสีมืดพรีเมียม (DARK: `#060d1f`), สีครีมสว่าง (CREAM: `#f4efe6`), และสีเหลืองมะนาว (LIME: `#c8ff00`)
*   **ระบบฟอนต์ (Typography)**:
    *   `Barlow Condensed` (หัวข้อ Display ตัวใหญ่)
    *   `DM Sans` (เนื้อหาหลักทั่วไป)
    *   `JetBrains Mono` (ข้อมูลย่อย ปุ่ม สเตตัส และโค้ด)
*   **โครงสร้างหน้าเว็บ (Layout Blocks)**:
    *   **ส่วนบน (Hero Section)**: โทนสีมืดพร้อมเส้นตารางกริดบาง ๆ (Decorative Grid) แสดงชื่อสะกดพิมพ์ใหญ่ ปรับเอฟเฟกต์ปุ่มและการเลื่อนลง
    *   **ตัวกั้นคลื่น (Wave Divider)**: เส้นหยัก SVG ที่ส่งต่อโทนเข้มลงสู่พื้นหลังสีครีมสว่าง
    *   **ส่วนทักษะ (Skills & Stack)**: เปลี่ยนเป็นกริดตารางบาง 4 คอลัมน์ที่คมและมีรายละเอียดชัดเจน
    *   **ส่วนผลงาน (Projects Grid)**: ปรับดีไซน์การ์ดผลงานแบบคมเฉียบ (Sharp borders - `rounded-none`) มีปุ่มกดขยายดูรายละเอียดแท็กสแต็คที่เป็นพื้นหลังสีดำตัวหนังสือมะนาว และลิ้งก์เปิดดูตัวอย่าง
    *   **สเตตัสและฟอร์มติดต่อ (Status & Contact Form)**: ปรับช่องป้อนและปุ่มส่งข้อความให้เรียบหรู คม ปราศจากส่วนโค้งกลม
*   **แก้ปัญหาความเสถียรบนระบบ Windows**: ย้ายการนำเข้าไอคอนจาก `lucide-react` ไปสร้างและเรียกใช้งานผ่านไฟล์ SVG จำลอง [Icons.tsx](file:///c:/Users/BANK/Desktop/Personal/src/components/Icons.tsx) แบบตรงเพื่อเลี่ยงบั๊กคอมไพเลอร์ SWC บนเครื่องพัฒนา Windows ทำให้ระบบรันได้เสถียร 100%

## 📝 2. คู่มือการแก้ไขและอัปเดตเนื้อหาในภายหลัง (Future Content Updates)

หากคุณต้องการอัปเดตเนื้อหา เช่น การเพิ่มผลงานใหม่, การเปลี่ยนรายละเอียดตัวตน หรือการแก้สแต็คความชำนาญ สามารถเปิดไฟล์ต่าง ๆ และทำตามคำแนะนำด้านล่างนี้ได้เลยครับ:

### 👤 A. การแก้ไขข้อมูลส่วนตัวของคุณ (ชื่อ, บทบาท, คำอธิบายตัวตน, ลิงก์โซเชียล)
*   **ไฟล์ที่ต้องแก้ไข**: [src/data/projects.ts](file:///c:/Users/BANK/Desktop/Personal/src/data/projects.ts) ที่ตัวแปร `PROFILE`
*   **วิธีแก้ไข**: แก้ไขข้อความในฟิลด์ที่กำหนด เช่น:
    ```typescript
    export const PROFILE: Profile = {
      name: "ชื่อของคุณ",
      role: "ตำแหน่งสายงานของคุณ",
      tagline: "ประโยคสั้นแนะนำตัวที่จะแสดงใน Hero หน้าแรก",
      about: "คำอธิบายตัวตนของคุณแบบยาว",
      yearsExperience: "3+ years", // จำนวนปีประสบการณ์
      focus: "หมวดเน้นย้ำ เช่น Web apps · Infrastructure",
      resumeUrl: "/resume.pdf", // ลิงก์ดาวน์โหลดประวัติ
      social: [
        { label: "linkedin.com/in/...", url: "ลิงก์ LinkedIn ของคุณ" },
        { label: "github.com/...", url: "ลิงก์ GitHub ของคุณ" },
      ],
    };
    ```

### 💼 B. การเพิ่ม/แก้ไขผลงาน (Projects)
*   **วิธีที่ 1 (ใช้ระบบเขียนร่างผ่านหน้าเว็บ - แนะนำ)**:
    1.  เปิดเว็บในเบราว์เซอร์ของคุณ (`http://localhost:3000`)
    2.  เลื่อนไปที่ส่วนของ Projects ด้านล่างสุดจะมีปุ่มสีดำชื่อ **`+ Draft New Project`**
    3.  เมื่อกดแล้ว ให้กรอกข้อมูล ชื่อผลงาน, สแต็คเทคโนโลยี, ลิงก์ Demo หรือ GitHub และรายละเอียดโปรเจกต์
    4.  กด **`Publish Draft`** เพื่อส่งข้อมูลเข้าไปบันทึกจำลองในระบบชั่วคราว
    5.  ระบบจะสร้างหน้าต่างแจ้งเตือนพร้อมสัญลักษณ์ **`✓`** และสร้างโค้ดภาษา TypeScript ออกมาให้คุณโดยเฉพาะ
    6.  **ให้ทำทำการก๊อปปี้โค้ดในกรอบนั้น** แล้วนำไปวางต่อด้านหน้าสุดของอาร์เรย์ `PROJECTS` ในไฟล์ [src/data/projects.ts](file:///c:/Users/BANK/Desktop/Personal/src/data/projects.ts)
*   **วิธีที่ 2 (เพิ่มข้อมูลโค้ดโดยตรง)**:
    *   **ไฟล์ที่ต้องแก้ไข**: [src/data/projects.ts](file:///c:/Users/BANK/Desktop/Personal/src/data/projects.ts) ที่ตัวแปร `PROJECTS`
    *   **วิธีแก้ไข**: เพิ่มโครงสร้างโปรเจกต์ใหม่ลงไปในอาร์เรย์ตามรูปแบบดังนี้:
        ```typescript
        export const PROJECTS: Project[] = [
          {
            title: "ชื่อโปรเจกต์ใหม่ของคุณ",
            summary: "คำอธิบายรายละเอียดแบบย่อเกี่ยวกับโปรเจกต์นี้",
            role: "ตำแหน่ง/บทบาทของคุณในการทำโปรเจกต์นี้",
            tech: ["Next.js", "TypeScript", "Tailwind CSS"], // ลิสสแต็คที่ใช้
            url: "https://demo-link.com", // ลิงก์เว็บจริง (ถ้าไม่มีให้ลบฟิลด์นี้ออกได้)
            repoUrl: "https://github.com/...", // ลิงก์ GitHub (ถ้าไม่มีให้ลบฟิลด์นี้ออกได้)
          },
          // ... โปรเจกต์เดิมที่มีอยู่
        ];
        ```

## 💻 3. คำสั่งสำหรับการทำงาน (Commands)

ใช้คำสั่งเหล่านี้ใน Terminal ของคุณที่โฟลเดอร์โปรเจกต์เพื่อเปิดทดสอบหรือจัดเตรียมไฟล์ขึ้นเซิร์ฟเวอร์จริง:

*   **คำสั่งรันทดสอบบนเครื่อง (Development Mode)**:
    ```bash
    npm run dev
    ```
    *(จากนั้นเปิดเว็บเบราว์เซอร์ไปที่ลิงก์ `http://localhost:3000`)*

*   **คำสั่งบิวด์ระบบเพื่อขึ้นโปรดักชัน (Production Build)**:
    ```bash
    npm run build
    ```
